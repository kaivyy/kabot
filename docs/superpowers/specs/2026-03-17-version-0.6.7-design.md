# Version 0.6.7 Release Docs Update Design

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this design.

**Goal:** Update Kabot release artifacts and documentation to version 0.6.7 with consistent release notes.

**Architecture:** Bump the authoritative package version, align CLI-visible version strings, and update human-facing release notes (CHANGELOG, README, MkDocs releases page) with the latest changes. Keep historical release plans unchanged.

**Tech Stack:** Python packaging via `pyproject.toml`, Markdown docs (README, CHANGELOG, MkDocs site docs).

---

## Scope

### In-scope updates
- Package version bump to **0.6.7**.
- Update release notes in CHANGELOG, README, and MkDocs releases.
- Keep version display consistent across CLI surfaces (via `kabot.__version__`).

### Out of scope
- Historical plan documents in `docs/plans/…`.
- Sub-skill package versions (e.g., `kabot/skills/local-places`).
- Release automation beyond docs/version strings.
