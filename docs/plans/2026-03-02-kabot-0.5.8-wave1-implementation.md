# Program 0.5.8 - Wave 1 Implementation Notes

Date: 2026-03-02
Scope: Stability + Ops hardening (Wave 1), with onboarding/trust foundations for Wave 3

## Delivered in this iteration

1. Config schema extensions (typed, backward-compatible):
- `runtime.observability`
  - `enabled`
  - `emit_structured_events`
  - `sample_rate`
  - `redact_secrets`
- `runtime.quotas`
  - `enabled`
  - `max_cost_per_day_usd`
  - `max_tokens_per_hour`
  - `enforcement_mode`
- `security.trust_mode`
  - `enabled`
  - `verify_skill_manifest`
  - `allowed_signers`
- `skills.onboarding`
  - `auto_prompt_env`
  - `auto_enable_after_install`
  - `soul_injection_mode`

2. Config migration/default injection:
- Loader now injects defaults for:
  - `runtime.observability`
  - `runtime.quotas`
  - `security.trustMode`
  - `skills.onboarding`
- Skills canonicalization is always normalized through migration path (with existing atomic backup behavior).

3. `kabot doctor --parity-report`:
- Added CLI flag and parity renderer.
- Added machine-readable export mode:
  - `--parity-json <path>` writes report to disk.
  - `--parity-json -` prints raw JSON to stdout.
- Added parity diagnostic payload with mandatory sections:
  - `runtime_resilience`
  - `fallback_state_machine`
  - `adapter_registry`
  - `migration_status`
  - `bridge_health`
  - `skills_precedence`

4. Runtime observability markers:
- Structured runtime events are now emitted in:
  - turn start/end + context build
  - LLM attempt and attempt result
  - tool idempotency hits

5. Runtime quota guardrail (warn/hard):
- Added per-turn quota checks in fallback execution path.
- `warn` mode logs warning but continues.
- `hard` mode blocks request with clear quota error.

6. Skill trust-mode enforcement in CLI installer:
- `kabot skills install --git ...` now validates signer trust policy when `security.trust_mode.enabled=true`.
- Installation is blocked when manifest verification fails or signer is not trusted.
- Installed skill can be auto-enabled via `skills.onboarding.auto_enable_after_install`.
7. Skill one-shot onboarding in CLI installer:
- `kabot skills install --git ...` now also applies onboarding flow:
  - optional env prompting for required keys (`auto_prompt_env`),
  - optional persona injection into workspace `SOUL.md` (`soul_injection_mode`),
  - canonical `skills` normalization before save for stable config shape.
8. Soak gate foundation:
- Added alpha soak gate evaluator (`kabot/utils/soak_gate.py`) and integrated it into parity diagnostics.
- `kabot doctor --parity-report` now shows `soak_gate` status from `~/.kabot/logs/soak_latest.json` when available.
