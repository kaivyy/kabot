# v0.6.4 Skill-First Routing And Search Fallback Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Reduce parser-heavy routing so Kabot prefers skills first, only uses legacy finance parsers as fallback, and can pivot from unavailable/empty `web_search` to `web_fetch` when the request is actually a direct-page fetch.

**Architecture:** We will tighten deterministic routing at the intent layer instead of deleting whole subsystems in one sweep. Legacy stock/crypto tools stay registered but become explicit fallback lanes. Search behavior becomes more honest and more useful by distinguishing search queries from page-fetch queries, then letting runtime short-circuit into `web_fetch` when the user is really asking for a site/page rather than a generic search. After behavior changes we cut `v0.6.4` by updating metadata, docs, changelog, and release artifacts.

**Tech Stack:** Python 3.11, pytest, MkDocs, Hatch build, GitHub Actions, PyPI.
